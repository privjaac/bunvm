Essential Commands

Everything you need to manage Bun versions

bunvm install <version>

Install a specific version of Bun

$ bunvm install latest
$ bunvm install 1.0.25
$ bunvm install 1.0.0
bunvm use <version>

Switch to a specific version (persists across shells)

$ bunvm use 1.0.25
 Ahora usando Bun 1.0.25
9 Versiï¿½n guardada como default para futuros shells
bunvm list [--local]

List available or installed versions

$ bunvm list --local
 Version    | Status       | Use | Install Path
--------------------------------------------------------------------------------
 1.0.25     | installed*   | *   | ~/.bunvm/versions/bun-1.0.25
 1.0.24     | installed    |     | ~/.bunvm/versions/bun-1.0.24
bunvm alias <name> <version>

Create memorable aliases for versions

$ bunvm alias stable 1.0.25
$ bunvm alias beta 1.1.0
$ bunvm use stable
bunvm current

Show currently active version

$ bunvm current
 Versiï¿½n actual: 1.0.25
bunvm selfupdate

Update BunVM to the latest version

$ bunvm selfupdate
 BunVM actualizado correctamente.

Multiple Projects

Different projects, different Bun versions. No manual switching needed.

# Project A
$ cd ~/projects/project-a
$ echo "1.0.25" > .bun-version
# Auto-switches to 1.0.25! <ï¿½

# Project B
$ cd ~/projects/project-b
$ echo "1.0.20" > .bun-version
# Auto-switches to 1.0.20! <ï¿½
>ï¿½

Testing Beta Features

Test new features without breaking your workflow.

# Setup versions
$ bunvm install 1.0.25
$ bunvm alias stable 1.0.25
$ bunvm install 1.1.0
$ bunvm alias beta 1.1.0

# Easy switching
$ bunvm use beta
# Test features...
$ bunvm use stable

Team Collaboration

Ensure everyone uses the same Bun version.

# In your project
$ echo "1.0.25" > .bun-version
$ git add .bun-version
$ git commit -m "Pin Bun version"

# Team members:
$ git clone <repo>
$ cd <repo>
# BunVM auto-detects version! >
